Hexo + Butterfly 博客环境配置全流程（新手保姆级）

作为一个刚入门博客搭建的新手，折腾了一段时间，终于完成了 Hexo + Butterfly 主题的全部配置，从本地调试到连接 GitHub Pages，再到个性化设置，踩了不少小坑，也总结了一套完整的流程。这篇文章就记录下我整个环境配置的全过程，希望能帮到和我一样的新手朋友，少走弯路～

一、前期准备（必备基础）

在开始配置之前，需要先准备好以下工具和环境，缺一不可：

  • 电脑环境：Windows/macOS 均可（本文以 Windows 为例）

  • 必备软件：Node.js（用于运行 Hexo）、Git（用于部署到 GitHub Pages）

  • 账号：GitHub 账号（用于创建博客仓库，实现外网访问）

  • 核心框架：Hexo（静态博客生成器）、Butterfly（博客主题，简洁美观，新手友好）

补充：Node.js 和 Git 安装直接官网下载，下一步下一步即可，无需复杂配置，安装完成后可通过终端输入 node -v 和 git --version 验证是否安装成功。

二、Hexo 基础环境搭建（本地博客初始化）

这一步主要完成本地博客的创建，让博客能在本地正常预览。

  1. 打开终端（Windows 用 CMD 或 PowerShell，macOS 用终端），输入命令安装 Hexo 脚手架：
    npm install -g hexo-cli

  2. 创建博客文件夹（自定义名称，比如 blog），进入文件夹：
    hexo init blog
    cd blog

  3. 安装依赖包：
    npm install

  4. 启动本地预览，验证是否搭建成功：
    hexo clean（清理缓存，可选）
    hexo generate（生成静态文件）
    hexo server（启动本地服务器）

  5. 打开浏览器，访问 http://localhost:4000，能看到 Hexo 默认页面，说明本地博客搭建成功。

三、Butterfly 主题安装与基础配置

Hexo 默认主题比较简单，我选择了 Butterfly 主题（颜值高、配置灵活），安装和基础配置步骤如下：

  1. 在博客根目录（my-blog）下，输入命令安装 Butterfly 主题：
    git clone https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly

  2. 修改 Hexo 配置文件（根目录下的 \_config.yml），将主题改为 butterfly：
    找到 theme: 一行，改为 theme: butterfly

  3. 重启本地预览（hexo clean && hexo s），此时页面会变成 Butterfly 主题的默认样式，基础主题配置完成。

五、连接 GitHub Pages（实现外网访问）

本地配置完成后，需要将博客部署到 GitHub Pages，这样全世界都能通过网址访问你的博客。

  1. 在 GitHub 上创建仓库：仓库名必须为 你的GitHub用户名\.github\.io（比如我的用户名为 test，仓库名就是 test.github.io），直接创建即可，无需初始化 README。

  2. 安装 Hexo 部署插件：在博客根目录终端输入：
    npm install hexo-deployer-git --save

六、常见问题与踩坑总结

  • 问题1：找不到 themes/butterfly/source/css/\_custom\.styl 文件？
    解决：新版 Butterfly 已移除该文件，直接使用 inject 注入样式和脚本即可（本文已提供完整配置）。

  • 问题2：本地图片不显示？
    解决：图片必须放入 source/img 文件夹，路径写 /img/图片名\.后缀，不要加 source。

  • 问题3：部署到 GitHub 后，访问 404？
    解决：仓库名必须为 用户名\.github\.io，部署分支为 main，部署后等待几分钟再访问。

七、总结

整个配置过程从本地搭建到外网部署，再到个性化设置，虽然踩了一些小坑，但整体难度不大，核心步骤就是：Hexo 初始化 → 安装 Butterfly 主题 → 个性化配置（菜单、头像、背景） → 部署到 GitHub Pages。
